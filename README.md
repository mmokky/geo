# GEO

Программный комплекс предназначен для камеральной обработки результатов полевых геодезических измерений: автоматического расчета ведомостей высотных и тахеометрических ходов, вычисления обратных геодезических задач (ОГЗ), автоматического определения дирекционных углов ориентирования станций, а также ведения каталогов опорных пунктов и тахеометрии с сохранением сессий в формате JSON.

## 🚀 Основные возможности

* **Управление проектами:** Создание, сохранение, импорт и экспорт изолированных сессий объектов в файлы конфигурации.
* **Каталог опорных точек:** Хранение исходных координат пунктов ($X, Y, H$), автоматическое вычисление дирекционных углов направлений.
* **Журнал высотных ходов:** Учет измерений по станциям (КЛ, КП), вычисление места нуля (МО), углов наклона ($\nu$), длин линий, приборов и целей. Поддерживается строгий контроль актуальности уравнивания (`traverse_is_calculated`).
* **Тахеометрическая съемка:** Динамическое формирование пикетов, выравнивание таблиц полевых журналов «на лету» с адаптивным расчетом приращений.
* **Гибкая точность:** Настройка округления плановых координат и высотных отметок от 0 до 4 знаков после запятой.
* **Логирование:** Автоматическое дублирование всех выведенных ведомостей и таблиц в текстовые файлы логов (`tacheometry_log_*.txt`) в кодировке UTF-8.

## 💻 Инструкция по установке и запуску

Программа кроссплатформенная и может быть запущена на любой популярной операционной системе.

### 1. Windows

Для Windows подготовлен автономный исполняемый файл, не требующий установки Python.

* **Запуск через готовый `.exe`:**
1. Скачайте архив с релизом и извлеките файл `geo.exe` (или `main.exe`).
2. Перенесите в удобную рабочую папку.
3. Запустите двойным щелчком мыши.


* **Запуск через Терминал (альтернативный):**
```bash
python3 main.py

```



### 2. macOS

Для macOS доступно скомпилированное приложение `.app`.

* **Запуск готового `.app`:**
1. Перенесите `geo.app` в папку `/Applications` («Программы»).
2. При первом запуске ОС может заблокировать приложение из-за отсутствия подписи разработчика. Перейдите в **Системные настройки** -> **Конфиденциальность и безопасность** и нажмите **«Все равно открыть»**.


* **Запуск через Терминал (альтернативный):**
```bash
python3 main.py

```



### 3. Linux

Скрипт оптимизирован для работы в Linux-терминалах (поддерживается автоматическое определение и очистка экрана через `clear`).

1. Установите Python (если не установлен):
```bash
sudo pacman -S python      # Для Arch / EndeavourOS
sudo apt install python3   # Для Ubuntu / Debian

```


2. Откройте эмулятор терминала в директории со скриптом и запустите его:
```bash
python main.py

```



### 4. Android

Запуск осуществляется через эмулятор терминала **Termux** (рекомендуется скачивать версию с F-Droid).

1. Откройте Termux и обновите пакеты:
```bash
pkg update && pkg upgrade

```


2. Установите Python и инструменты работы с файловой системой:
```bash
pkg install python termux-tools

```


3. Дайте доступ к внутренней памяти телефона, чтобы перекинуть файл скрипта:
```bash
termux-setup-storage

```


4. Перейдите в папку со скриптом (например, если он лежит в Загрузках):
```bash
cd /путь-до-файла/

```


5. Запустите программу:
```bash
python main.py

```



### 5. iOS (iPhone / iPad)

Прямой запуск `.exe` или `.app` невозможен. Скрипт запускается через специализированные среды выполнения Python.

* **Способ 1: Среда Pythonista 3 (Платная, самый удобный интерфейс)**
1. Установите приложение **Pythonista 3** из App Store.
2. Импортируйте файл `main.py` в приложение через стандартное меню "Share" (Поделиться) или встроенный файловый менеджер.
3. Откройте скрипт внутри приложения и нажмите кнопку **Run** (Play).


* **Способ 2: Среда Pyto (Бесплатная/Условно-бесплатная)**
1. Установите приложение **Pyto - Python 3** из App Store.
2. Откройте встроенный файловый локатор, выберите ваш файл `main.py` из приложения «Файлы» (iCloud Drive / Память устройства).
3. Запустите его встроенным интерпретатором.

## 🛠 Руководство пользователя (Порядок работы)

При запуске программы открывается **Главное меню**. Рекомендуется соблюдать следующую технологическую последовательность выполнения вычислений:

### Шаг 1: Инициализация проекта (Пункт 8 или 9)

По умолчанию программа создает сессию `default_project`. Если вы хотите разграничить объекты:

* Нажмите `8` для выбора существующего или создания нового файла базы данных (`.json`).
* Нажмите `9`, если необходимо полностью очистить текущие таблицы и задать используемую марку прибора (например, *2Т30*, *3Т2КП*).

### Шаг 2: Ввод каталога опорных точек (Пункт 1)

Перед расчетом ходов необходимо внести исходные пункты геодезической основы.

1. Выберите пункт `1`.
2. Введите имя точки (имена регистрозависимы, например, `Rp1`, `Пп4`).
3. Задайте координаты X, Y и высоту H.

> 💡 Если точка уже существует, программа покажет текущие координаты в формате `[сейчас: ...]`, позволяя быстро отредактировать их или нажать Enter для сохранения без изменений.

### Шаг 3: Ввод журналов полевых измерений (Пункт 2)

Основной блок для ввода данных с полевой книжки.

1. **Данные станции:** Укажите имя станции (должно соответствовать точке из каталога) и высоту прибора $i$.
2. **Ориентирование:** Укажите имя точки ориентирования. Если обе точки есть в каталоге, программа автоматически решит ОГЗ и высчитает исходный дирекционный угол.
3. **Измерения на пикеты/точки хода:** Для каждой визирной цели последовательно вводятся:
* Направление (Круг Лево / Круг Право) в градусах и минутах.
* Расстояние по дальномеру (или приведенное).
* Высота цели (вехи/рейки) $v$.



### Шаг 4: Расчет и уравнивание (Пункт 3 и 4)

* После ввода полевых данных выберите пункт `3`. Программа выполнит расчет места нуля (МО), углов наклона, вычислит невязки и переведет внутренний флаг `traverse_is_calculated` в состояние `True`.
* Выберите пункт `4`, чтобы вывести на экран **Полную уравненную ведомость хода** с итоговыми координатами.

> ⚠️ **Важно:** Если вы вернетесь в пункты 1 или 2 и измените/удалите хотя бы одну точку или станцию, система автоматически сбросит флаг уравнивания. Пункт 4 станет временно недоступен до тех пор, пока вы снова не запустите расчет в пункте 3.

### Шаг 5: Тахеометрическая съемка (Пункт 5)

Используется для обработки съемочных пикетов. Позволяет просмотреть структурированную таблицу тахеометрии с рассчитанными горизонтальными проложениями, превышениями и отметками пикетов.

## 🗂 Структура данных и резервное копирование

Все данные сохраняются автоматически при любом изменении в файл с расширением `.json` в директории программы. Пример структуры файла проекта:

```json
{
  "coord_digits": 2,
  "height_digits": 2,
  "catalog_points": {
    "Т1": {"x": 1500.45, "y": 2300.80, "h": 120.15}
  },
  "instrument": "2Т30",
  "measurements": [...],
  "piket_global_counter": 5,
  "tacheometry_data": {...},
  "traverse_is_calculated": true
}

```

Для резервного копирования данных вашего объекта достаточно скопировать соответствующий `.json` файл. При переносе программы на другое устройство просто положите этот файл рядом со скриптом/файлом запуска, и он появится в списке доступных проектов в пункте `8`.
